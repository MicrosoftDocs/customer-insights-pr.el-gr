# <a name="whats-new-in-the-audience-insights-capability-of-dynamics-365-customer-insights"></a>Τι νέο υπάρχει στη δυνατότητα πληροφοριών κοινού του Dynamics 365 Customer Insights

[!INCLUDE [cc-data-platform-banner](../includes/cc-data-platform-banner.md)]

Σας ανακοινώνουμε με χαρά τις πιο πρόσφατες ενημερωμένες εκδόσεις! Αυτό το άρθρο συνοψίζει τις δυνατότητες της δημόσιας έκδοσης προεπισκόπησης, τις γενικές βελτιώσεις διαθεσιμότητας και τις ενημερώσεις δυνατοτήτων. Για να δείτε τα μακροπρόθεσμα σχέδια δυνατοτήτων, ρίξτε μια ματιά στο θέμα [Σχέδια κυκλοφορίας του Dynamics 365 και του Power Platform](/dynamics365/release-plans/).

Επίσης, μπορείτε να παρακολουθήσετε το παρακάτω βίντεο για να μάθετε περισσότερα σχετικά με τις δυνατότητες που έχουν προγραμματιστεί για τους τελευταίους έξι μήνες.

> [!VIDEO https://www.youtube.com/embed/jQh-7pscH30]

Αναλαμβάνουμε την ανάπτυξη ενημερώσεων σε βάση ανά περιοχή. Έτσι, ορισμένες περιοχές μπορεί να δουν τις δυνατότητες πριν από άλλες. Εάν δεν καθοριστεί διαφορετικά, δεν χρειάζεται να κάνετε καμία ενέργεια και θα ενημερώσουμε την εφαρμογή αυτόματα χωρίς διακοπές λειτουργίας.

> [!TIP]
> ια υποβολή και ψήφο σε αιτήματα δυνατοτήτων και προτάσεις προϊόντων, μεταβείτε στην [πύλη ιδεών εφαρμογής Dynamics 365](https://experience.dynamics.com/ideas/categories/?forum=79a8c474-4e35-e911-a971-000d3a4f3343&forumName=Dynamics%20365%20Customer%20Insights).

## <a name="february-2021-updates"></a>Ενημερώσεις Φεβρουαρίου 2021

Οι ενημερώσεις του Φεβρουαρίου 2021 περιλαμβάνουν διάφορες δυνατότητες, αναβαθμίσεις επιδόσεων και επιδιορθώσεις σφαλμάτων.

#### <a name="extensibility"></a>Επεκτασιμότητα

- **Εξαγωγή τμημάτων στο AdRoll**

  Έχουμε επεκτείνει τους προορισμούς εξαγωγής για να συμπεριλάβουμε το AdRoll. Τώρα μπορείτε να εξαγάγετε τμήματα σε κοινά από το Customer Insights στο AdRoll και να τα χρησιμοποιήσετε ως γραμμή βάσης για τις διαφημίσεις σας. Για περισσότερες πληροφορίες, ανατρέξτε στη [Σύνδεση για AdRoll](export-adroll.md).

#### <a name="segments"></a>Τμήματα
 
- **Δημιουργία διπλοτύπου ενός τμήματος**
  
  Για να δημιουργήσετε ένα νέο τμήμα με βάση ένα υπάρχον, μπορείτε τώρα να δημιουργήσετε διπλότυπο τμήματος και να επεξεργαστείτε το διπλότυπο τμήμα για να το βελτιώσετε περαιτέρω. 

- **Προσθήκη επιπλέον χαρακτηριστικών σε τμήμα**

  Τώρα μπορείτε να συμπεριλάβετε χαρακτηριστικά στην έξοδο του τμήματός σας, ακόμα και αν αυτά τα χαρακτηριστικά δεν αποτελούν μέρος του προφίλ πελάτη. Για παράδειγμα, συμπεριλάβετε αναγνωριστικά συνδρομής σε ένα τμήμα, παρόλο που είναι μέρος της οντότητας συνδρομής που έχει σχέση M:1 με την οντότητα πελάτη. Εφόσον το χαρακτηριστικό ανήκει σε μια οντότητα που σχετίζεται με την οντότητα πελάτη, μπορείτε τώρα να συμπεριλάβετε αυτά τα χαρακτηριστικά.  

#### <a name="predictions"></a>Προβλέψεις

- **Δημιουργία προγνωστικών προτάσεων προϊόντων**

  Η κατανόηση του τι ενδιαφέρονται οι πελάτες να αγοράσουν είναι ένα από τα πρώτα βήματα που απαιτούνται για τη βελτίωση των επιχειρηματικών εσόδων και την ενίσχυση της αφοσίωσης των πελατών μέσω της εξατομίκευσης και της δέσμευσης. Η παροχή προτάσεων για προϊόντα που δεν είναι στοιχισμένα με τα ενδιαφέροντο του πελάτη σας μπορεί να δημιουργήσει μια αίσθηση αποσύνδεσης μεταξύ του πελάτη και της επιχείρησής σας και, εντέλει, να περιορίσει τα συνολικά πιθανά έσοδα και εμπειρία για έναν πελάτη. 

  Χρησιμοποιώντας τα δικά σας δεδομένα, μπορείτε πλέον να δημιουργήσετε προβλέψεις για τα προϊόντα που είναι πιθανό να αγοράσουν οι πελάτες σας στο μέλλον. Για περισσότερες πληροφορίες, ανατρέξτε στην [πρόβλεψη πρότασης προϊόντων](predict-product-recommendation.md).

#### <a name="system-administration"></a>Διαχείριση συστήματος

- **Το περιβάλλον αντιγραφής υποστηρίζει περισσότερους τύπους προέλευσης δεδομένων**

  Οι διαχειριστές μπορούν να αντιγράφουν ρυθμίσεις παραμέτρων περιβάλλοντος σε ένα νέο περιβάλλον στον ίδιο οργανισμό. Αυτή η δυνατότητα επεκτείνει τη λειτουργικότητα του περιβάλλοντος αντιγραφής για περιπτώσεις στις οποίες χρησιμοποιούνται προελεύσεις δεδομένων με βάση μια λίμνη δεδομένων Common Data Service ή έναν φάκελο Common Data Model.

## <a name="january-2021-updates"></a>Ενημερώσεις Ιανουαρίου 2021

Οι ενημερώσεις του Ιανουαρίου 2021 περιλαμβάνουν διάφορες δυνατότητες, αναβαθμίσεις επιδόσεων και επιδιορθώσεις σφαλμάτων.

#### <a name="extensibility"></a>Επεκτασιμότητα

- **Εκτεταμένη λειτουργικότητα και βελτιωμένη απόδοση για εξαγωγή SFTP** Μπορείτε τώρα να εξαγάγετε όλες τις οντότητες εξόδου από το Customer Insights σε έναν κεντρικό υπολογιστή SFTP. Προηγουμένως, οι εξαγωγές περιορίζονταν σε οντότητες του τμήματος. Επιπλέον, η απόδοση της εξαγωγής SFTP επιτρέπει μεγαλύτερο όγκο δεδομένων σε λιγότερο χρόνο, ανάλογα με την απόδοση του κεντρικού υπολογιστή SFTP.    
  Για περισσότερες πληροφορίες, ανατρέξτε στη [Σύνδεση για SFTP (προεπισκόπηση)](export-sftp.md).  

#### <a name="segments"></a>Τμήματα

- **Προτεινόμενα τμήματα που διαθέτουν εκμάθηση μηχανής για τη βελτίωση των μεγεθών** Υπάρχει ένας νέος τρόπος για να ανακαλύψετε και να δημιουργήσετε τμήματα. Το σύστημα χρησιμοποιεί ένα μοντέλο AI για να προτείνει τμήματα που μπορούν να βοηθήσουν στη βελτίωση ενός KPI (δείκτη) που παρακολουθείτε ήδη. Δείχνουμε την έκταση της επιρροής των χαρακτηριστικών που επιλέγετε σε ένα μέγεθος ή σε άλλο πρωτεύον χαρακτηριστικό. Αυτές οι πληροφορίες βοηθούν στην εύρεση πιθανών τμημάτων που παρουσιάζουν ευκαιρίες.    
  Για περισσότερες πληροφορίες, ανατρέξτε στα [Προτεινόμενα τμήματα (προεπισκόπηση)](suggested-segments.md).

#### <a name="data-unification"></a>Ενοποίηση δεδομένων

- **Βελτιωμένη εμπειρία αντιστοίχισης** Στον τομέα ενοποίησης δεδομένων, η εμπειρία αντιστοίχισης ενημερώθηκε. Σας επιτρέπει να ρυθμίσετε τις παραμέτρους και να προβάλετε τους κανόνες αντιστοίχισης, συμπεριλαμβανομένων λεπτομερών στατιστικών στοιχείων για να εξηγήσετε περαιτέρω πώς λειτουργεί η αντιστοίχιση. Υπάρχουν επιλογές για την απενεργοποίηση ενός κανόνα αντιστοίχισης, ώστε να μην είναι πλέον ενεργός κατά τη διατήρηση των κανόνων ρύθμισης παραμέτρων, μεταφοράς και απόθεσης και πολλών άλλων.
  Για περισσότερες πληροφορίες, ανατρέξτε στην ενότητα [Αντιστοίχιση οντοτήτων](match-entities.md).

- **Η έξοδος κατάργησης διπλοτύπων από τη διαδικασία αντιστοίχισης είναι διαθέσιμη ως οντότητα** Η έξοδος της διαδικασίας κατάργησης διπλοτύπων από τη διαδικασία αντιστοίχισης τώρα γράφεται σε ξεχωριστή οντότητα για περαιτέρω ανάλυση. Αυτή η οντότητα αποτελείται από τα πεδία που χρησιμοποιούνται στη διαδικασία κατάργησης διπλοτύπων και το αρχείο νικητή και τις αντίστοιχες εναλλακτικές εγγραφές που συγχωνεύονται με το αρχείο νικητή.
  Για περισσότερες πληροφορίες, ανατρέξτε στο θέμα [Έξοδος κατάργησης διπλοτύπων ως οντότητα](match-entities.md#deduplication-output-as-an-entity).

#### <a name="system-administration"></a>Διαχείριση συστήματος

- **Απρόσκοπτη κοινή χρήση δεδομένων στο Microsoft Dataverse** Μπορείτε τώρα να μοιραστείτε την έξοδο του Customer Insights με εφαρμογές Microsoft Dataverse που χρησιμοποιούν το Microsoft Dataverse Managed Data Lake. Μόλις συσχετίσετε ένα περιβάλλον Dataverse με το Customer Insights, έχετε την επιλογή να ενεργοποιήσετε την κοινή χρήση δεδομένων.
  Για περισσότερες πληροφορίες, ανατρέξτε στην ενότητα [Διαχείριση περιβαλλόντων](manage-environments.md).

  Επιπλέον, προσθέσαμε νέες επιλογές διαμόρφωσης για να ελέγξουμε καλύτερα τον τρόπο με τον οποίο θέλετε να εμπλουτίζονται τα προφίλ των πελατών σας.

- **Έλεγχος του ποια προφίλ θα εμπλουτίζονται**

  Τώρα μπορείτε να εμπλουτίσετε μόνο ένα υποσύνολο των προφίλ πελατών σας με την επιλογή να επιλέξετε μια οντότητα τμήματος αντί για την προεπιλεγμένη οντότητα πελάτη. Δημιουργήστε ένα τμήμα με τα προφίλ πελατών που θέλετε να εμπλουτίσετε και επιλέξτε το στη ρύθμιση παραμέτρων εμπλουτισμού για το σύνολο δεδομένων των πελατών σας.

  Για περισσότερες πληροφορίες, ανατρέξτε στο θέμα [Εμπλουτισμός προφίλ πελατών με δημογραφικά στοιχεία από την Experian](enrichment-experian.md) ή [Εμπλουτισμός προφίλ πελατών με τη HERE Technologies](enrichment-here.md).

### <a name="new-and-updated-features-in-november-2020"></a>Νέες και ενημερωμένες δυνατότητες του Νοεμβρίου 2020

Μάθετε περισσότερα σχετικά με τον εμπλουτισμό με HERE Technologies.

  Για περισσότερες πληροφορίες, ανατρέξτε στο θέμα [Εμπλουτισμός προφίλ πελατών με HERE Technologies](enrichment-here.md).

  Για περισσότερες πληροφορίες, ανατρέξτε στην ενότητα [Σύνδεση με έναν φάκελο Common Data Model](connect-common-data-model.md#connect-to-a-common-data-model-folder).

#### <a name="predictions"></a>Προβλέψεις

- **Πρόβλεψη απώλειας συναλλαγών**

  Η δυνατότητα πρόβλεψης απώλειας συναλλαγών σάς δίνει τη δυνατότητα, χωρίς τη βοήθεια ενός επιστήμονα δεδομένων, να προβλέψετε την πιθανότητα ένας πελάτη να σταματήσει την αγορά προϊόντων ή υπηρεσιών.  Χρησιμοποιώντας τη βαθμολογία πρόβλεψης, μπορείτε να συνδυάσετε άλλες πληροφορίες σχετικά με τους πελάτες σας, όπως η τιμή του πελάτη, για να δημιουργήσετε τμήματα υψηλού κινδύνου απώλειας ή πελάτες υψηλής αξίας. Χρησιμοποιήστε αυτό το τμήμα για να στοχεύσετε απευθείας στους πελάτες μέσω δραστηριοτήτων μάρκετινγκ, υποστήριξης πελατών και άλλων σεναρίων για τη μείωση του κινδύνου απώλειας.
 
  Ρυθμίστε τις παραμέτρους του ορισμού της απώλειας ως παράθυρο βάσει χρόνου ειδικά για την επιχείρησή σας και καθορίστε πότε οι πελάτες θα θεωρείται ότι έχουν χαθεί. Για παράδειγμα, ένα παντοπωλείο μπορεί να θέλει να θεωρεί ότι ένας πελάτης χάθηκε σε περίπτωση που δεν έχει αγοράσει τίποτα τις τελευταίες 30 ημέρες.
 
  Καθώς συνεχίζετε να δημιουργείτε το πρόβλεψη, θα σας καθοδηγήσουμε ποια δεδομένα απαιτούνται και θα σας δώσουμε τη δυνατότητα να αντιστοιχίσετε δεδομένα σχετικά με την επιχείρησή σας σε πεδία που απαιτούνται για την πρόβλεψη της απώλειας για τους πελάτες σας. Χρησιμοποιώντας τη βαθμολογία πρόβλεψης, μπορείτε να συνδυάσετε άλλες πληροφορίες σχετικά με τους πελάτες σας για να δημιουργήσετε τμήματα υψηλού κινδύνου απώλειας. Με τη βοήθεια αυτού του τμήματος, μπορείτε να στοχεύσετε απευθείας στους πελάτες στο μάρκετινγκ, στην εξυπηρέτηση πελατών και σε άλλα σενάρια για να μειώσετε τον κίνδυνο απώλειας για συγκεκριμένους πελάτες για τη βελτίωση των εσόδων και τη μείωση του κόστους.

  Εντός της εμπειρίας, μπορείτε να ρυθμίσετε τον ορισμό της απώλειας ως παράθυρο βάσεις χρόνου ειδικά αναφορικά με την επιχείρησή σας. Για παράδειγμα, μια επιχείρηση ροής βίντεο που έχει μια διεργασία μηνιαίας συνδρομής μπορεί να θελήσει να θεωρήσει ένα πελάτης χαμένο μετά από 15 ημέρες μετά τη λήξη της συνδρομής του.

  Καθώς συνεχίζετε διαμέσου της πρόβλεψης, θα σας καθοδηγήσουμε σχετικά με το είδος των δεδομένων που απαιτούνται και θα σας δώσουμε τη δυνατότητα να αντιστοιχίσετε δεδομένα σχετικά με την επιχείρησή σας σε πεδία που απαιτούνται για την πρόβλεψη των απωλειών για τους πελάτες σας. Καθώς αλλάζουν οι επιχειρηματικές πληροφορίες, μπορείτε επίσης να ορίσετε ένα χρονοδιάγραμμα για επανακατάρτιση με βάση νέες πληροφορίες στο σύστημά σας, ώστε να προσαρμόζονται στις μεταβαλλόμενες επιχειρηματικές συνθήκες.    
  Για περισσότερες πληροφορίες, ανατρέξτε στην ενότητα [Πρόβλεψη απώλειας συνδρομής (προεπισκόπηση)](predict-subscription-churn.md).

#### <a name="segments"></a>Τμήματα

- **Εύρεση παρόμοιων πελατών**
  
  Εύρεση παρόμοιων πελατών στη βάση πελατών σας με χρήση τεχνητής νοημοσύνης. Ένα μοντέλο εκμάθησης μηχανής δυαδικής ταξινόμησης αντιστοιχίζει μια βαθμολογία ομοιότητας σε πελάτες στο αναπτυγμένο τμήμα. Η βαθμολογία βασίζεται στην ομοιότητα με τους πελάτες στο τμήμα προέλευσης. Ανάλογα με τη βαθμολογία ομοιότητας, τα προφίλ πελατών προστίθενται σε ένα πρόσφατα δημιουργημένο τμήμα.

  Μερικές φορές αναφέρεται ως μοντελοποίηση όμοιων στο ψηφιακό μάρκετινγκ και χρησιμοποιεί ένα μοντέλο AI για να βοηθήσει στην εύρεση πελατών που είναι παρόμοιοι με ένα άλλο τμήμα των πελατών σας, λαμβάνοντας υπόψη περισσότερα χαρακτηριστικά. Δεν σας επιτρέπει μόνο να επιλέξετε τα χαρακτηριστικά αλλά σας επιτρέπει επίσης να καθορίσετε το μέγιστο αριθμό πελατών που θα πρέπει να βρίσκονται σε αυτό το νέο τμήμα. Στη συνέχεια το μοντέλο AI θα υπολογίσει τα αποτελέσματα της ομοιότητας για κάθε πελάτη με βάση τα επιλεγμένα χαρακτηριστικά σας και θα βρει πελάτες με το υψηλότερο μέσο αποτέλεσμα ομοιότητας. Το τμήμα που προκύπτει θα περιλαμβάνει πελάτες που μοιάζουν με τον πελάτη στο αρχικό σας τμήμα.    
  Για περισσότερες πληροφορίες, ανατρέξτε στην ενότητα [Παρόμοιοι πελάτες](find-similar-customer-segments.md).

- **Επικάλυψη τμημάτων και παράγοντες διαφοροποίησης**

  Η επικάλυψη τμημάτων σάς δίνει τη δυνατότητα να δείτε πόσοι και ποιοι πελάτες που είναι κοινοί σε δύο ή περισσότερα τμήματα. Για παράδειγμα, ο τρόπος με τον οποίο ένα τμήμα πελατών υψηλών δαπανών επικαλύπτεται με ένα τμήμα πελατών με υψηλή ικανοποίηση ή με τον τρόπο που ένα τμήμα πελατών που χάνονται επικαλύπτεται με ένα τμήμα πελατών χαμηλής ικανοποίησης. Επιπλέον, μπορείτε να αναλύσετε τον τρόπο με τον οποίο αλλάζει η επικάλυψη με βάση ένα επιπλέον χαρακτηριστικό της επιλογής σας.

  Οι διαφοροποιήσεις τμημάτων αποκαλύπτουν αυτό που διαφοροποιεί ένα τμήμα από τους υπόλοιπους πελάτες σας ή από κάποιο άλλο τμήμα. Το μόνο που χρειάζεται να κάνετε είναι να αναγνωρίσετε ένα τμήμα αγοράς και το σύστημα θα προσδιορίσει τα χαρακτηριστικά και τα μέτρα προφίλ που διακρίνουν το τμήμα αυτό, στη μορφή μιας ταξινομημένης λίστας παραγόντων διαφοροποίησης, από τον ισχυρότερο στη ασθενέστερο παράγοντα.    
  Για περισσότερες πληροφορίες, δείτε [Πληροφορίες τμήματος (προεπισκόπηση)](segment-insights.md).

Μια νέα εμπειρία τάσης εισάγεται για τη γρήγορη αναγνώριση σημαντικών αλλαγών σε τμήματα.

Χωρίς την πλήρη εικόνα των αλληλεπιδράσεων του κάθε πολίτη σε όλα τα κανάλια, οι κυβερνήσεις δεν μπορούν να εκσυγχρονιστούν να εκσυγχρονιστεί σε κλίμακα. Η Microsoft δεσμεύεται να στηρίζει τις τεχνολογικές ανάγκες της κυβέρνησης ώστε να συμβαδίζει με τις προσδοκίες των πολιτών για συνεπή και δυναμική εμπειρία.    
  Με το κύμα 1 της έκδοσης 2020, το Dynamics 365 Customer Insights θα είναι διαθέσιμο για το Government Community Cloud (GCC), ένα περιβάλλον που κατασκευάζεται για να καλύψει τις υψηλότερες ανάγκες συμμόρφωσης των κυβερνητικών υπηρεσιών των Ηνωμένων Πολιτειών. Οι οργανισμοί αποκτούν μια ενοποιημένη άποψη των πολιτών και χρησιμοποιούν το προκατασκευασμένο AI για να αντλήσουν ιδέες που βελτιώνουν τις αλληλεπιδράσεις, ενδυναμώνουν τους υπαλλήλους και μετατρέπουν τις κοινότητες, μειώνοντας την πολυπλοκότητα IT και πληρούν τα πρότυπα συμμόρφωσης και ασφάλειας των Ηνωμένων Πολιτειών.   Οι χρήστες μπορούν να δημιουργήσουν, να επεξεργαστούν και να διαγράψουν μετρήσεις που βασίζονται σε άλλες μετρήσεις. Για παράδειγμα, μια μέτρηση που έχει κατασκευαστεί με βάση μια άλλη μέτρηση, η οποία έχει κατασκευαστεί με βάση μια τρίτη μέτρηση.

#### <a name="segments"></a>Τμήματα

- **Ένας άλλος τελεστής**
  
  Ο εντός συνόλου τελεστής επιτρέπει την τμηματοποίηση για τους πελάτες με διάφορες πιθανές τιμές συμβολοσειράς. Πριν από την προσθήκη αυτού του τελεστή, πρέπει να δημιουργήσετε αυτά τα τμήματα με πολλαπλές συνθήκες OR. Ο εντός συνόλου τελεστής σάς επιτρέπει να το κάνετε αυτό με μία μόνο συνθήκη.    
  Για περισσότερες πληροφορίες, ανατρέξτε στο θέμα [Δημιουργία και διαχείριση τμημάτων](segments.md).

#### <a name="system-administration"></a>Διαχείριση συστήματος

- **Αντιγραφή ρυθμίσεων ρύθμισης παραμέτρων σε ένα νέο περιβάλλον**
  
  Αντιγράψτε τις ρυθμίσεις παραμέτρων από ένα περιβάλλον σε ένα άλλο. Κατά τη δημιουργία ενός νέου περιβάλλοντος, μπορείτε να επιλέξετε ένα υπάρχον περιβάλλον από το οποίο θέλετε να αντιγράψετε τη ρύθμιση παραμέτρων. Προς το παρόν, υποστηρίζουμε προελεύσεις δεδομένων, ενοποίηση δεδομένων, σχέσεις, μετρήσεις και τμήματα που πρόκειται να αντιγραφούν. Δεν αντιγράφονται τα διαπιστευτήρια προέλευσης δεδομεων και τα πραγματικά δεδομένα.    
  Για περισσότερες πληροφορίες, ανατρέξτε στην ενότητα [Διαχείριση περιβαλλόντων](manage-environments.md).
