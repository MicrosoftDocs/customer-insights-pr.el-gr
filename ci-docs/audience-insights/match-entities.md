# <a name="match-entities"></a>Αντιστοίχιση οντοτήτων

Αφού ολοκληρώσετε τη φάση του χάρτη, είστε έτοιμοι να αντιστοιχίσετε τις οντότητες σας. Η φάση της αντιστοίχισης προσδιορίζει τον τρόπο συνδυασμού των συνόλων δεδομένων σας σε ένα ενοποιημένο σύνολο δεδομένων προφίλ πελάτη. Η φάση της αντιστοιχίας απαιτεί τουλάχιστον [δύο αντιστοιχισμένες οντότητες](map-entities.md).

## <a name="specify-the-match-order"></a>Καθορίστε τη σειρά αντιστοιχίας

Μεταβείτε στην ενότητα **Ενοποίηση** > **Αντιστοιχία** και επιλέξτε **Ορισμός σειράς** για να ξεκινήσετε τη φάση της αντιστοιχίας.

Κάθε αντιστοιχία ενοποιεί δύο ή περισσότερες οντότητες σε μία οντότητα, ενώ διατηρεί σταθερή κάθε καρτέλα μοναδικού πελάτη. Στο παρακάτω παράδειγμα επιλέξαμε τρεις οντότητες: **ContactCSV: TestData** ως **Κύρια** οντότητα, **WebAccountCSV: TestData** ως **Οντότητα 2** και **CallRecordSmall: TestData** ως **Οντότητα 3**. Το διάγραμμα πάνω από τις επιλογές απεικονίζει τον τρόπο με τον οποίο θα εκτελεστεί η σειρά αντιστοιχίας.

> [!div class="mx-imgBorder"]
> ![Επεξεργασία της σειράς αντιστοίχισης δεδομένων](media/configure-data-match-order-edit-page.png "Επεξεργασία της σειράς αντιστοίχισης δεδομένων")
  
Η **Κύρια** οντότητα είναι αντιστοιχισμένη με την **Οντότητα 2**. Το σύνολο δεδομένων που προκύπτει από την πρώτη αντιστοίχιση αντιστοιχίζεται με την **Οντότητα 3**.
Σε αυτό το παράδειγμα, επιλέξαμε μόνο δύο αντιστοιχίες, αλλά το σύστημα μπορεί να υποστηρίξει περισσότερες.

> [!IMPORTANT]
> Η οντότητα που επιλέγετε ως την **Κύρια** οντότητα θα χρησιμεύσει ως βάση για τον ενοποιημένο κύριο σύνολο δεδομένων σας. Οι πρόσθετες οντότητες που επιλέγονται κατά τη φάση της αντιστοιχίας θα προστεθούν σε αυτήν την οντότητα. Ταυτόχρονα, αυτό δεν σημαίνει ότι η ενοποιημένη οντότητα θα συμπεριλάβει *όλα* τα δεδομένα που περιλαμβάνονται σε αυτήν την οντότητα.
>
> Υπάρχουν δύο θέματα που μπορούν να σας βοηθήσουν να επιλέξετε την ιεραρχία των οντοτήτων σας:
>
> - Ποια οντότητα θεωρείτε ότι διαθέτει τα πληρέστερα και πιο αξιόπιστα δεδομένα σχετικά με τους πελάτες σας;
> - Η οντότητα που μόλις προσδιορίσατε έχει χαρακτηριστικά τα οποία χρησιμοποιούνται από κοινού και από άλλες οντότητες (για παράδειγμα, το όνομα, ο αριθμός τηλεφώνου ή η διεύθυνση ηλεκτρονικού ταχυδρομείου); Εάν όχι, επιλέξτε τη δεύτερη πιο αξιόπιστη οντότητα.

Επιλέξτε **Τέλος** για να αποθηκεύσετε την σειρά αντιστοιχίας σας.

## <a name="define-rules-for-your-first-match-pair"></a>Καθορίστε κανόνες για το πρώτο σας ζεύγος αντιστοιχίας

Αφού καθορίσετε τη σειρά αντιστοιχίας, θα δείτε τις καθοριζόμενες αντιστοιχίες στη σελίδα **Αντιστοιχία**. Τα πλακίδια στο επάνω μέρος της οθόνης θα είναι κενά μέχρι να εκτελεστεί η σειρά αντιστοιχίας.

> [!div class="mx-imgBorder"]
> ![Καθορισμός κανόνων](media/configure-data-match-need-rules.png "Καθορισμός κανόνων")

Η προειδοποίηση **Χρειάζεται κανόνες** υποδεικνύει ότι δεν ορίστηκε κανένας κανόνας αντιστοίχισης για ένα ζεύγος αντιστοίχισης. Οι κανόνες αντιστοίχισης καθορίζουν τη λογική με την οποία θα αντιστοιχιστεί ένα συγκεκριμένο ζεύγος οντοτήτων.

1. Για να καθορίσετε τον πρώτο σας κανόνα, ανοίξτε το παράθυρο **Ορισμός κανόνων** επιλέγοντας την αντίστοιχη γραμμή αντιστοιχίας στον πίνακα αντιστοιχίσεων (1) και στη συνέχεια, επιλέξτε **Δημιουργία νέου κανόνα** (2).

   > [!div class="mx-imgBorder"]
   > ![Δημιουργία νέου κανόνα](media/configure-data-match-new-rule2.png "Δημιουργία νέου κανόνα")

2. Στο τμήμα παραθύρου **Επεξεργασία κανόνα**, ρυθμίστε τις συνθήκες για αυτόν τον κανόνα. Κάθε συνθήκη αντιπροσωπεύεται από δύο γραμμές που περιλαμβάνουν υποχρεωτικές επιλογές.

   > [!div class="mx-imgBorder"]
   > ![Τμήμα παραθύρου νέου κανόνα](media/configure-data-match-new-rule-condition.png "Τμήμα παραθύρου νέου κανόνα")

   Οντότητα/πεδίο (πρώτο) - Ένα χαρακτηριστικό που θα χρησιμοποιηθεί για την αντιστοιχία από την οντότητα του πρώτου ζεύγους αντιστοιχίας. Τέτοια παραδείγματα μπορεί να είναι ένας αριθμός τηλεφώνου ή μια διεύθυνση ηλεκτρονικού ταχυδρομείου. Επιλέξτε ένα χαρακτηριστικό που ενδέχεται να είναι μοναδικό για τον πελάτη.

   > [!TIP]
   > Αποφύγετε την αντιστοιχία με βάση χαρακτηριστικά τύπου δραστηριότητας. Με άλλα λόγια, εάν ένα χαρακτηριστικό φαίνεται να είναι δραστηριότητα, τότε μπορεί να είναι ένα ακατάλληλο κριτήριο για αντιστοιχία.  

   Οντότητα/πεδίο (Δεύτερο) - Ένα χαρακτηριστικό που θα χρησιμοποιηθεί για την αντιστοιχία από την οντότητα του δεύτερου ζεύγους αντιστοιχίας.

   Κανονικοποίηση **Μέθοδος κανονικοποίησης**: Διατίθενται διάφορες επιλογές κανονικοποίησης για τα επιλεγμένα χαρακτηριστικά. Για παράδειγμα, κατάργηση σημείων στίξης ή κατάργηση διαστημάτων

   Για την κανονικοποίηση ονομάτων οργανισμών (προεπισκόπηση), μπορείτε επίσης να επιλέξετε **Τύπο (Τηλέφωνο, Όνομα, Οργανισμός)**

   > [!div class="mx-imgBorder"]
   > ![Κανονικοποίηση-B2B](media/match-normalization-b2b.png "Κανονικοποίηση-B2B")

   Επίπεδο ακρίβειας - Το επίπεδο ακρίβειας που θα χρησιμοποιηθεί για αυτήν την συνθήκη. Ο ορισμός ενός επιπέδου ακρίβειας για μια συνθήκη αντιστοιχίας μπορεί να είναι δύο τύπων: **Βασικό** και **Προσαρμοσμένο**.  
   - Βασικό: Σας παρέχει τέσσερις επιλογές ανάμεσα στις οποίες μπορείτε να επιλέξετε: Χαμηλό, Μεσαίο, Υψηλό και Ακριβές. Επιλέξτε **Ακριβές** για να αντιστοιχίσετε μόνο καρτέλες που ταιριάζουν 100 τοις εκατό. Επιλέξτε ένα από τα άλλα επίπεδα για να ταιριάξετε καρτέλες που δεν είναι 100 τοις εκατό ταυτόσημες.
   - Προσαρμοσμένο: Χρησιμοποιήστε το ρυθμιστικό για να καθορίσετε το προσαρμοσμένο ποσοστό που πρέπει να διαθέτουν οι καρτέλες ή εισαγάγετε μια τιμή στο **Προσαρμοσμένο** πεδίο. Το σύστημα θα αντιστοιχίσει μόνο καρτέλες που περνούν αυτό το όριο, όπως ζεύγη αντιστοίχισης που έχουν συνδυαστεί. Οι τιμές στο ρυθμιστικό είναι μεταξύ 0 και 1. Επομένως το 0,64 αντιπροσωπεύει το 64 τοις εκατό.

3. Για να αποθηκεύσετε τον κανόνα, κάντε κλικ στο κουμπί **Τέλος**.

### <a name="add-multiple-conditions"></a>Προσθήκη πολλών συνθηκών

Για να αντιστοιχίσετε τις οντότητες σας μόνο εάν πληρούνται πολλές συνθήκες, προσθέστε και άλλες συνθήκες που συνδέονται μέσω ενός τελεστή AND.

1. Στο παράθυρο **Επεξεργασία κανόνα**, επιλέξτε **Προσθήκη συνθήκης**. Μπορείτε, επίσης, να καταργήσετε συνθήκες επιλέγοντας το κουμπί κατάργησης που βρίσκεται δίπλα σε μια υπάρχουσα συνθήκη.

2. Για να αποθηκεύσετε τον κανόνα κάντε κλικ στο κουμπί **Τέλος**.

## <a name="add-multiple-rules"></a>Προσθήκη πολλών κανόνων

Κάθε συνθήκη ισχύει για ένα μόνο ζεύγος χαρακτηριστικών ενώ οι κανόνες αντιπροσωπεύουν σύνολα συνθηκών. Για να αντιστοιχίσετε τις οντότητες σας με διαφορετικά σύνολα χαρακτηριστικών, μπορείτε να προσθέσετε περισσότερους κανόνες.

1. Στις πληροφορίες κοινού, μεταβείτε στα **Δεδομένα** > **Ενοποίηση** > **Αντιστοίχιση**.

2. Επιλέξτε την οντότητα που θέλετε να ενημερώσετε και επιλέξτε **Προσθήκη κανόνων**.

3. Ακολουθήστε τη διαδικασία που περιγράφεται στο [Ορισμός κανόνων για το πρώτο ζεύγος αντιστοιχίας](#define-rules-for-your-first-match-pair).

> [!NOTE]
> Η σειρά των κανόνων έχει σημασία. Ο αλγόριθμος αντιστοίχησης προσπαθεί να κάνει τις αντιστοιχίες με βάση τον πρώτο κανόνα σας και συνεχίζει στο δεύτερο κανόνα μόνο εάν δεν εντοπίστηκαν αντιστοιχίες σύμφωνα με τον πρώτο κανόνα.

## <a name="define-deduplication-on-a-match-entity"></a>Ορισμός κατάργησης διπλοτύπων σε μια οντότητα αντιστοίχισης

Μαζί με τον καθορισμό κανόνων αντιστοίχισης μεταξύ οντοτήτων, όπως περιγράφεται στις παραπάνω ενότητες, μπορείτε επίσης να καθορίσετε τους κανόνες κατάργησης διπλοτύπων. *Η κατάργηση διπλοτύπων* είναι μια διεργασία. Προσδιορίζει τις διπλότυπες καρτέλες, τις συγχωνεύει σε μία καρτέλα και συνδέει όλες τις καρτέλες προέλευσης σε αυτήν τη συγχωνευμένη καρτέλα με εναλλακτικά αναγνωριστικά στη συγχωνευμένη καρτέλα.

Μετά την ταυτοποίηση μιας καρτέλας που έχει αναπαραχθεί, αυτή η καρτέλα θα χρησιμοποιηθεί στη διεργασία αντιστοίχισης μεταξύ οντοτήτων. Η κατάργηση διπλοτύπων υλοποιείται σε επίπεδο οντότητας και μπορεί να εφαρμοστεί σε κάθε οντότητα που χρησιμοποιείται στη διεργασία αντιστοίχισης.

### <a name="add-deduplication-rules"></a>Προσθήκη κανόνων κατάργησης διπλοτύπων

1. Στις πληροφορίες κοινού, μεταβείτε στα **Δεδομένα** > **Ενοποίηση** > **Αντιστοίχιση**.

1. Στην ενότητα **Συγχωνευμένα διπλότυπα**, επιλέξτε **Ορισμός οντοτήτων**.

1. Στην ενότητα **Προτιμήσεις συγχώνευσης**, επιλέξτε τις οντότητες στις οποίες θέλετε να εφαρμόσετε κατάργηση διπλοτύπων.

1. Καθορίστε τον τρόπο συγχώνευσης των διπλότυπων καρτελών και επιλέξτε μία από τις τρεις επιλογές συγχώνευσης:
   - *Οι πιο γεμάτες*: προσδιορίζει την καρτέλα με τα περισσότερο γεμάτα χαρακτηριστικά ως την καρτέλα του νικητή. Αυτή είναι η προεπιλεγμένη επιλογή συγχώνευσης.
   - *Πιο πρόσφατη*: προσδιορίζει την καρτέλα του νικητή με βάση τις πλέον πρόσφατες. Απαιτεί μια ημερομηνία ή ένα αριθμητικό πεδίο για τον καθορισμό του πρόσφατου χαρακτήρα.
   - *Λιγότερο πρόσφατη*: προσδιορίζει την καρτέλα του νικητή με βάση τις λιγότερο πρόσφατες. Απαιτεί μια ημερομηνία ή ένα αριθμητικό πεδίο για τον καθορισμό του πρόσφατου χαρακτήρα.
 
   > [!div class="mx-imgBorder"]
   > ![Βήμα 1 κανόνων κατάργησης διπλοτύπων](media/match-selfconflation.png "Βήμα 1 κανόνων κατάργησης διπλοτύπων")
 
1. Αφού επιλέξετε τις οντότητες και οριστεί η προτίμηση συγχώνευσης, επιλέξτε **Δημιουργία νέου κανόνα** για να καθορίσετε τους κανόνες κατάργησης διπλοτύπων σε ένα επίπεδο οντότητας.
   - **Επιλέξτε πεδίο** παρουσιάζει όλα τα διαθέσιμα πεδία από την εν λόγω οντότητα στην οποία θέλετε να καταργήσετε διπλότυπα δεδομένα προέλευσης.
   - Καθορίστε τις ρυθμίσεις κανονικοποίησης και ακρίβειας με τον ίδιο τρόπο όπως καθορίζεται στην αντιστοίχιση μεταξύ οντοτήτων.
   - Μπορείτε να καθορίσετε πρόσθετες συνθήκες επιλέγοντας **Προσθήκη συνθήκης**.
 
   > [!div class="mx-imgBorder"]
   > ![Βήμα 2 κανόνων κατάργησης διπλοτύπων](media/match-selfconflation-rules.png "Βήμα 2 κανόνων κατάργησης διπλοτύπων")

  Μπορείτε να δημιουργήσετε πολλούς κανόνες κατάργησης διπλοτύπων για μια οντότητα. 

1. Η εκτέλεση της διαδικασίας αντιστοίχισης ομαδοποιεί πλέον τις καρτέλες με βάση τις συνθήκες που καθορίζονται στους κανόνες κατάργησης διπλοτύπων. Αφού ομαδοποιήσετε τις καρτέλες, η πολιτική συγχώνευσης εφαρμόζεται για τον προσδιορισμό της καρτέλας του νικητή.

1. Αυτή η καρτέλα νικητή μεταβιβάζεται στη συνέχεια στην αντιστοίχιση μεταξύ οντοτήτων.

1. Τυχόν προσαρμοσμένοι κανόνες αντιστοίχισης που έχουν οριστεί για την αντιστοίχιση πάντα και ποτέ δεν συμφωνούν με τους κανόνες κατάργησης διπλοτύπων. Εάν ένας κανόνας κατάργησης διπλοτύπων προσδιορίζει τις καρτέλες που ταιριάζουν και ένας προσαρμοσμένος κανόνας αντιστοίχισης έχει ρυθμιστεί ώστε να μην ταιριάζει ποτέ με αυτές τις καρτέλες, τότε αυτές οι δύο καρτέλες δεν θα αντιστοιχιστούν.

1. Μετά την εκτέλεση της διεργασίας αντιστοίχισης, θα δείτε τα στατιστικά κατάργησης διπλοτύπων.
   
> [!NOTE]
> Ο καθορισμός κανόνων κατάργησης διπλοτύπων δεν είναι υποχρεωτικός. Εάν δεν έχουν ρυθμιστεί τέτοιοι κανόνες, εφαρμόζονται οι κανόνες που ορίζονται από το σύστημα. Συμπτύσσουν όλες τις καρτέλες που μοιράζονται τον ίδιο συνδυασμό τιμών (ακριβής αντιστοιχία) από το πρωτεύον κλειδί και τα πεδία των κανόνων που ταιριάζουν σε μια μεμονωμένη καρτέλα πριν μεταβιβάσει τα δεδομένα οντότητας σε αντιστοίχιση μεταξύ οντοτήτων για βελτιωμένη απόδοση και λογική συστήματος.

## <a name="run-your-match-order"></a>Εκτέλεση της σειράς αντιστοιχίας σας

Αφού καθορίσετε τους κανόνες αντιστοίχισης, συμπεριλαμβανομένων των κανόνων αντιστοίχισης και κατάργησης διπλοτύπων μεταξύ οντοτήτων, μπορείτε να πραγματοποιήσετε την εκτέλεση της σειράς αντιστοιχίας. Στη σελίδα **Αντιστοιχία**, επιλέξτε **Εκτέλεση** για να ξεκινήσετε τη διεργασία. Μπορεί να χρειαστεί αρκετός χρόνος για να ολοκληρωθεί ο αλγόριθμος αντιστοιχίας. Δεν μπορείτε να αλλάξετε τις ιδιότητες στη σελίδα **Αντιστοιχία** μέχρι να ολοκληρωθεί η διαδικασία αντιστοιχίας. Θα βρείτε την οντότητα ενοποιημένου προφίλ πελάτη που δημιουργήθηκε στη σελίδα **Οντότητες**. Η ενοποιημένη οντότητα του πελάτη σας ονομάζεται **ConflationMatchPairs : CustomerInsights**.

Για να κάνετε πρόσθετες αλλαγές και να εκτελέσετε ξανά το βήμα, μπορείτε να ακυρώσετε μια αντιστοίχιση σε εξέλιξη. Επιλέξτε το κείμενο **Γίνεται ανανέωση...** και επιλέξτε **Ακύρωση εργασίας** στο κάτω μέρος του πλαϊνού παραθύρου που εμφανίζεται.

Όταν ολοκληρωθεί η διαδικασία αντιστοίχισης, το κείμενο **Γίνεται ανανέωση...** θα αλλάξει σε **Επιτυχία** και μπορείτε να χρησιμοποιήσετε ξανά όλες τις λειτουργίες της σελίδας.

Η πρώτη διεργασία αντιστοιχίας έχει ως αποτέλεσμα τη δημιουργία μιας ενοποιημένης κύριας οντότητας. Όλες οι επόμενες διεργασίες εκτελούνται ως αποτέλεσμα της επέκτασης της εν λόγω οντότητας.

> [!TIP]
> Υπάρχουν [έξι τύποι κατάστασης](system.md#status-types) για εργασίες/διεργασίες. Επιπλέον, οι περισσότερες διεργασίες [εξαρτώνται από άλλες διεργασίες κατάντη](system.md#refresh-policies). Μπορείτε να επιλέξετε την κατάσταση μιας διεργασίας για να δείτε λεπτομέρειες σχετικά με την πρόοδο ολόκληρου του έργου. Αφού επιλέξετε **Προβολή λεπτομερειών** για μία από τις εργασίες του έργου, μπορείτε να βρείτε πρόσθετες πληροφορίες: χρόνος επεξεργασίας, ημερομηνία τελευταίας επεξεργασίας και όλα τα σφάλματα και τις προειδοποιήσεις που σχετίζονται με την εργασία.

## <a name="review-and-validate-your-matches"></a>Εξετάστε και επικυρώστε τις αντιστοιχίες σας

Αξιολογήστε την ποιότητα των ζευγών αντιστοιχίας σας και βελτιώστε την:

- Στη σελίδα **Αντιστοιχία**, θα βρείτε δύο πλακίδια που θα εμφανίζουν τις αρχικές πληροφορίες σχετικά με τα δεδομένα σας.

  - **Μοναδικοί πελάτες**: εμφανίζει τον αριθμό των μοναδικών προφίλ που εντόπισε το σύστημα.
  - **Αντιστοιχισμένες καρτέλες**: εμφανίζει τον αριθμό των αντιστοιχιών σε όλα τα ζεύγη αντιστοιχίας σας.

- Στον πίνακα **Σειρά αντιστοιχίας**, μπορείτε να αξιολογήσετε τα αποτελέσματα κάθε ζεύγους αντιστοιχίας, συγκρίνοντας τον αριθμό των καρτελών που προέρχονται από αυτήν την οντότητα ζεύγους αντιστοιχίας με το ποσοστό των καρτελών που έχουν αντιστοιχιστεί με επιτυχία.

- Στην ενότητα **Κανόνες** μιας οντότητας στον πίνακα **Σειρά αντιστοιχίας"**, θα βρείτε το ποσοστό των καρτελών που έχουν αντιστοιχιστεί με επιτυχία σε επίπεδο κανόνα. Επιλέγοντας το σύμβολο του πίνακα δίπλα σε έναν κανόνα, μπορείτε να προβάλετε όλες αυτές τις καρτέλες στο επίπεδο του κανόνα. Συνιστούμε να ελέγξετε ένα υποσύνολο των καρτελών για να επαληθεύσετε ότι αντιστοιχίζονται σωστά.

- Πειραματιστείτε με διαφορετικά όρια ακρίβειας γύρω από τις συνθήκες σας για να προσδιορίσετε τη βέλτιστη τιμή.

  1. Επιλέξτε τις τρεις τελείες (...) για τον κανόνα αντιστοιχίας ζεύγους με τον οποίο θέλετε να πειραματιστείτε και επιλέξτε **Επεξεργασία**.

  2. Επιλέξτε τη συνθήκη με την οποία θέλετε να πειραματιστείτε. Κάθε κριτήριο αντιπροσωπεύεται από μία γραμμή στο τμήμα παραθύρου **Αντιστοιχία κανόνα**.

  3. Αυτό που θα δείτε στη σελίδα **Προεπισκόπηση κριτηρίων** εξαρτάται από το επίπεδο ακρίβειας που έχετε επιλέξει για μια συνθήκη. Βρείτε τον αριθμό των αντιστοιχισμένων και των μη αντιστοιχισμένων καρτελών για την επιλεγμένη συνθήκη.

     Αποκτήστε μια άρτια κατανόηση των αποτελεσμάτων των διαφόρων επιπέδων κατωφλίου. Μπορείτε να συγκρίνετε τον αριθμό των καρτελών που θα αντιστοιχιστούν σε κάθε ένα από τα επίπεδα κατωφλίου και να προβάλετε τις καρτέλες κάτω από κάθε επιλογή. Επιλέξτε καθένα από τα πλακίδια και εξετάστε τα δεδομένα στην ενότητα του πίνακα.

## <a name="optimize-your-matches"></a>Βελτιστοποιήστε τις αντιστοιχίες σας

Αυξήστε την ποιότητα με την εκ νέου διαμόρφωση παραμέτρων ορισμένων από τις παραμέτρους αντιστοιχίας:

- **Αλλάξτε τη σειρά αντιστοιχίας** επιλέγοντας **Επεξεργασία** και αλλάξτε τα πεδία σειράς αντιστοιχίας.

  > [!div class="mx-imgBorder"]
  > ![Επεξεργασία σειράς αντιστοιχίας δεδομένων](media/configure-data-match-order-edit.png "Επεξεργασία σειράς αντιστοιχίας δεδομένων")

- **Αλλάξτε τη σειρά των κανόνων σας** εάν έχετε ορίσει πολλούς κανόνες. Μπορείτε να αλλάξετε τη σειρά των κανόνων αντιστοιχίας επιλέγοντας **Μετακίνηση προς τα πάνω** και **Μετακίνηση προς τα κάτω** στο πλέγμα κανόνων αντιστοιχίας.

  > [!div class="mx-imgBorder"]
  > ![Αλλαγή σειράς κανόνων](media/configure-data-change-rule-order.png "Αλλαγή σειράς κανόνων")

- **Αντιγράψτε τους κανόνες σας** εάν έχετε ορίσει έναν κανόνα αντιστοιχίας και θα θέλατε να δημιουργήσετε έναν παρόμοιο κανόνα με τροποποιήσεις. Αυτό γίνεται με την επιλογή **Διπλότυπο**.

  > [!div class="mx-imgBorder"]
  > ![Δημιουργία διπλότυπου κανόνα](media/configure-data-duplicate-rule.png "Δημιουργία διπλότυπου κανόνα")

- **Επεξεργαστείτε τους κανόνες σας** επιλέγοντας το σύμβολο **Επεξεργασία**. Μπορείτε να εφαρμόσετε κάνετε τις εξής αλλαγές:

  - Αλλαγή χαρακτηριστικών για μια συνθήκη: Επιλέξτε νέα χαρακτηριστικά εντός της συγκεκριμένης γραμμής συνθήκης.
  - Αλλάξτε το κατώφλι για μια συνθήκη: Προσαρμόστε το ρυθμιστικό ακριβείας.
  - Αλλάξτε τη μέθοδο κανονικοποίησης για μια συνθήκη: Ενημερώστε τη μέθοδο κανονικοποίησης.

## <a name="specify-your-custom-match-records"></a>Καθορίστε τις προσαρμοσμένες καρτέλες αντιστοίχισης

Μπορείτε να ορίσετε συνθήκες στις οποίες κάποιες μεμονωμένες καρτέλες θα πρέπει να ταιριάζουν πάντα ή να μην ταιριάζουν ποτέ. Οι κανόνες αυτοί μπορούν να αποσταλούν μαζικά στη διεργασία της αντιστοιχίας.

1. Επιλέξτε την επιλογή **Προσαρμοσμένη αντιστοιχία** στην οθόνη **Σειρά αντιστοιχίας**.

   > [!div class="mx-imgBorder"]
   > ![Δημιουργία προσαρμοσμένης αντιστοιχίας](media/custom-match-create.png "Δημιουργία προσαρμοσμένης αντιστοιχίας")

2. Εάν δεν έχετε μεταφορτωμένες οντότητες, θα δείτε ένα νέο παράθυρο διαλόγου **Προσαρμοσμένη αντιστοιχία** που απαιτεί τη συμπλήρωση ορισμένων λεπτομερειών. Εάν έχετε παράσχει αυτές τις λεπτομέρειες προηγουμένως, προχωρήστε στο βήμα 8.

   > [!div class="mx-imgBorder"]
   > ![Παράθυρο διαλόγου νέα προσαρμοσμένη αντιστοιχία](media/custom-match-new-dialog-box.png "Παράθυρο διαλόγου νέα προσαρμοσμένη αντιστοιχία")

3. Επιλέξτε **Συμπληρώστε το πρότυπο** για να λάβετε ένα αρχείο προτύπου που μπορεί να καθορίζει τις καρτέλες από τις οποίες οι οντότητες θα πρέπει να αντιστοιχούν πάντα ή να μην αντιστοιχούν ποτέ. Θα πρέπει να συμπληρώσετε χωριστά τις καρτέλες "αντιστοίχιση πάντοτε" και "αντιστοίχιση ποτέ" σε δύο διαφορετικά αρχεία.

4. Το πρότυπο περιέχει πεδία που καθορίζουν τις τιμές οντότητας και πρωτεύοντος κλειδιού οντότητας που θα χρησιμοποιηθούν στην προσαρμοσμένη αντιστοιχία. Για παράδειγμα, εάν θέλετε το πρωτεύον κλειδί 12345 από την οντότητα Πωλήσεις να είναι πάντα ίδιο με το πρωτεύον κλειδί 34567 από την οντότητα Επαφή, θα πρέπει να καθορίσετε τα εξής:
    - Entity1: Πωλήσεις
    - Entity1Key: 12345
    - Entity2: Επαφή
    - Entity2Key: 34567

   Το ίδιο αρχείο προτύπου μπορεί να καθορίσει προσαρμοσμένες καρτέλες αντιστοιχίας από πολλές οντότητες.

5. Αφού προσθέσετε όλες τις παρακάμψεις που θέλετε να εφαρμόσετε, αποθηκεύστε το αρχείο προτύπου.

6. Μεταβείτε στα **Δεδομένα** > **Προελεύσεις δεδομένων** και λάβετε τα αρχεία προτύπου ως νέες οντότητες. Μετά την πρόσληψή τους, μπορείτε να τα χρησιμοποιήσετε για να καθορίσετε τη ρύθμιση παραμέτρων της Αντιστοιχίας.

7. Αφού μεταφορτώσετε τα αρχεία και οι οντότητες είναι διαθέσιμες, επιλέξτε ξανά την επιλογή **Προσαρμοσμένη αντιστοιχία**. Θα δείτε επιλογές για τον καθορισμό των οντοτήτων που θέλετε να συμπεριλάβετε. Επιλέξτε τις απαιτούμενες οντότητες από το αναπτυσσόμενο μενού.

   > [!div class="mx-imgBorder"]
   > ![Προσαρμοσμένες παρακάμψεις αντιστοιχίας](media/custom-match-overrides.png "Προσαρμοσμένες παρακάμψεις αντιστοιχίας")

8. Επιλέξτε τις οντότητες που θέλετε να χρησιμοποιήσετε για **Πάντα αντιστοιχία** και **Ποτέ αντιστοιχία**, επιλέξτε **Τέλος**.

9. Επιλέξτε **Αποθήκευση** στη σελίδα **Αντιστοιχία** για την προσαρμοσμένη ρύθμιση παραμέτρων αντιστοιχίας που μόλις δημιουργήσατε.

10. Επιλέξτε **Εκτέλεση** στη σελίδα **Αντιστοιχία** για να ξεκινήσετε τη διεργασία αντιστοιχίας και η ρύθμιση παραμέτρων προσαρμοσμένης αντιστοιχίας θα εφαρμοστεί. Οι κανόνες που αντιστοιχίζονται στο σύστημα παρακάμπτονται από το σύνολο ρύθμισης παραμέτρων.

11. Μετά την ολοκλήρωση της αντιστοίχισης, μπορείτε να επαληθεύσετε την οντότητα **ConflationMatchPair** για να επιβεβαιώσετε ότι οι παρακάμψεις εφαρμόζονται στις αντιστοιχίσεις συνδυασμού.
